The Goal

Score the most points by cooperating or defecting in a multiplayer Iterated Prisoner's Dilemma tournament inspired by Axelrod's tournaments. Read here for inspiration on strategies.

The default bots randomly select from a set of classic strategies: Always Cooperate, Always Defect, Tit for Tat, Suspicious Tit for Tat, Tit for Two Tats, Random, Grudger, and Pavlov.

  Rules

Each game lasts 60 rounds with 2 to 8 players competing simultaneously.

Every round is independent. You will receive your opponent's choices last round as input for reference. For output. you choose to cooperate (C) or defect (D) against each active opponent. Your choices and your opponents' choices determine the points awarded.

Scoring per pairwise interaction:

  • Both cooperate (C vs C): 4 points each
  • Both defect (D vs D): 1 point each
  • Defector vs Cooperator (D vs C): 7 points for defector, 0 points for cooperator

Your score each round is the sum of all pairwise payoffs against active opponents.

You can download the source code of this game on GitHub.

Victory Conditions
  • If at any point only one player remains active, that player wins and the game ends immediately.
  • After 60 rounds, the player with the highest total score wins
Defeat Conditions
You lose if:
  • Your total score is lower than your opponents' after 60 rounds

You are eliminated if you:

  • Output the wrong number of lines
  • Use invalid output format
  • Reference yourself, non-existent, or eliminated players
  • Include duplicate opponent IDs
  • Use invalid moves (only C or D allowed)
  • Timeout

Eliminated players are removed from all opponent lists.

  Game Input/Output

Initialization input
Line 1: integer initialOpponentCount, the initial number of opponents.

Note: Your player ID is always 0. Opponents are numbered 1, 2, 3, etc.
Input for one game turn
Line 1: integer opponentCount, the number of active opponents this turn.
Next opponentCount lines: two space-separated values opponentId (integer) and move (character), where opponentId is the opponent's ID and move is their previous move against you (C for cooperate, D for defect, or N for no previous move on first turn).
Output for one game turn
opponentCount lines: two space-separated values opponentId move, where opponentId (integer) is the opponent's ID and move (character) is your move against them (C for cooperate or D for defect).
Constraints
2 to 8 players per game
60 rounds maximum
Response time per turn ≤ 100ms
